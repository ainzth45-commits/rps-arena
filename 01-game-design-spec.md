## 2. คำศัพท์ (ใช้ให้ตรงกันทั้งโปรเจกต์)

| คำ | ความหมาย |
|---|---|
| **มูฟ** | ค้อน / กรรไกร / กระดาษ — **ห้ามเรียกว่า "ไม้"** (เจ้าของเกมสั่ง) |
| **ชุดมูฟ (move set)** | มูฟ 3 ช่องที่ผู้เล่นตั้งไว้ล่วงหน้า: เป่า1 / เป่า2 / เป่า3 |
| **ตัวชี้** | ตัวบอกว่าครั้งหน้าที่ถูกท้า จะออกมูฟจากช่องไหน |
| **ผู้เล่น** | คนที่จ่ายเหรียญมาเล่นในรอบนั้น (ฝ่ายรุก เลือกมูฟเอง) |
| **ผู้ท้าชิง** | คนที่ถูกเลือกให้ดวลด้วย (ฝ่ายรับ ระบบออกมูฟแทน) |
| **รอบ** | สิทธิ์ที่ได้จากการจ่าย 3 เหรียญ 1 ครั้ง |
| **ซีซั่น** | ช่วงการแข่งขัน `SS1`, `SS2`, … ซุปเป็นคนกดจบ |
| **ลงสังเวียน** | ตั้งชุดมูฟของซีซั่นนี้เรียบร้อยแล้ว → คนอื่นท้าได้ |

## 3. ผู้เล่นและการลงทะเบียน

- ใช้ระบบเดียวกับเกมที่ 1: **ลงทะเบียนในแอป** ไม่ใส่ข้อมูลจริงลง source code
- ข้อมูลผู้เล่น: **รหัส** (ตัวพิมพ์ใหญ่ 1 + ตัวเลข 3 = 4 ตัวเป๊ะ ห้ามซ้ำ) · **ชื่อ** · **รูป** (ถ่ายกล้อง / เลือกไฟล์ / วางลิงก์ http)
- แก้ไขชื่อ/รูปได้ · **แก้รหัสไม่ได้** (รหัส = id ผูกข้อมูลทั้งหมด)
- **Guard การลบผู้เล่น:** ลบไม่ได้ถ้าคนนั้นกำลังอยู่ในรอบที่เปิดค้าง หรืออยู่ในดวลนอกรอบที่ยังไม่จบ

## 4. ซีซั่น

### 4.1 เปิดซีซั่นใหม่

1. คะแนนหลักของทุกคน = **30**
2. คะแนนรอง = **0** · สถิติทั้งหมด = 0 · ประวัติการดวลล้าง (ย้ายไปเก็บใน record ของซีซั่นเก่า)
3. **ล้างชุดมูฟของทุกคน** → ทุกคนต้องมาตั้งใหม่ (จงใจสร้าง "ช่วงเวลา relax" เป็นอีเวนต์เปิดซีซั่น)
4. ตัวชี้ของทุกคนกลับไปช่อง 1 โดยปริยาย

### 4.2 ลงสังเวียน

- **การตั้งชุดมูฟครั้งแรกของซีซั่น = ฟรี ไม่ต้องจ่ายเหรียญ**
- **คนที่ยังไม่ตั้งชุดมูฟของซีซั่นนี้ ไม่ปรากฏในรายการเลือกผู้ท้าชิง และไม่มีทางถูกสุ่มโดน**
- คนที่ยังไม่ลงสังเวียน **ยังเป็นผู้เล่นได้** (จ่ายเหรียญแล้วท้าคนอื่นได้) — แต่ระบบจะบังคับให้ตั้งชุดมูฟก่อนเข้าเมนูรอบ เพื่อความยุติธรรม (ไม่งั้นจะมีคนท้าคนอื่นได้ฝ่ายเดียวโดยไม่มีใครท้ากลับได้)

### 4.3 จบซีซั่น

1. ซุปกดจบ (มีป๊อปอัปยืนยัน) → **หน้าประกาศแชมป์**
2. บันทึก **record ถาวร**: อันดับทั้งหมด · คะแนน · สถิติ · ชุดมูฟสุดท้ายของทุกคน (เปิดเผยได้แล้วเพราะจบแล้ว)
3. record ดูย้อนหลังได้ทุกซีซั่น ไม่มีวันลบ
4. กด "เริ่มซีซั่นใหม่" → ทำตาม §4.1

## 5. ชุดมูฟและตัวชี้

### 5.1 ชุดมูฟ

- 3 ช่อง: **เป่า1 / เป่า2 / เป่า3** · แต่ละช่องเลือกได้จาก ค้อน / กรรไกร / กระดาษ
- **ซ้ำกันได้** (เช่น ค้อน / กรรไกร / กรรไกร)

### 5.2 ตัวชี้ — กลไกหัวใจของเกม

- ถูกท้าครั้งแรก → ระบบออก **เป่า1** → ตัวชี้เลื่อนไปช่อง 2
- ครั้งถัดไป **เป่า2** → **เป่า3** → **วนกลับเป่า1** ไปเรื่อยๆ
- **ตัวชี้เดินเมื่อถูกท้าเท่านั้น** ไม่สนว่าใครเป็นคนท้า ไม่สนว่าผลแพ้หรือชนะ
- **ตัวชี้ผูกกับตัวผู้ท้าชิง** ไม่ใช่ผูกกับคู่การดวล

### 5.3 การรีเซตตัวชี้

| การกระทำ | รีเซตไหม |
|---|---|
| กด **ยืนยัน** ในหน้าปรับชุดมูฟ (เปลี่ยนค่า) | ✅ กลับไปช่อง 1 |
| กด **ยืนยัน** ในหน้าปรับชุดมูฟ (ไม่เปลี่ยนอะไรเลย) | ✅ กลับไปช่อง 1 ← **"ซื้อการสับขาหลอก" ด้วย 3 เหรียญ** |
| เข้าหน้าปรับชุดมูฟแล้วกดยกเลิก/ถอยออก | ❌ ไม่รีเซต (ทบทวนชุดมูฟตัวเองได้ฟรี) |
| ตั้งชุดมูฟครั้งแรกของซีซั่น | ✅ เริ่มที่ช่อง 1 อยู่แล้ว |
| ถูกท้าในโหมดดวลนอกรอบ | ❌ ไม่เดิน ไม่รีเซต |

> **เหตุผลที่ต้องเป็นแบบนี้:** ตัวชี้ที่เดินเป็นระบบทำให้เซตถูกอ่านออกได้ใน 3 ครั้ง เกมจึงบังคับให้คนต้องจ่ายเหรียญมาสับขาหลอกเรื่อยๆ = ท่อยอดขายของเกม

## 6. รอบการเล่น

### 6.1 การเข้ารอบ

1. ผู้เล่นจ่าย **3 เหรียญ** ให้ซุปในโลกจริง (แอปไม่เก็บยอดเหรียญ)
2. ซุปกดเลือกชื่อคนนั้นจากหน้าแรก → เข้าสู่รอบของคนนั้น
3. จอแรกที่เจอเสมอ = **"📬 ระหว่างที่คุณไม่อยู่"**

### 6.2 สิ่งที่ทำได้ใน 1 รอบ

| ทำได้ | ครั้ง | หมายเหตุ |
|---|---|---|
| **ดวล** | 1 | เลือกทำก่อนหรือหลังปรับชุดมูฟก็ได้ · ข้ามได้ |
| **ยืนยันปรับชุดมูฟ** | 1 | เลือกทำก่อนหรือหลังดวลก็ได้ · ข้ามได้ |

### 6.3 การจบรอบ

- รอบจบเมื่อ **ดวลเสร็จ** (กลับหน้าแรกอัตโนมัติหลังดูผล) **หรือ** ซุปกดปุ่ม "จบรอบ"
- **เข้ามาปรับชุดมูฟอย่างเดียวไม่ดวล ก็ถือว่าใช้รอบไปแล้ว** (จ่ายเหรียญไปแล้ว)
- **Guard:** ถ้ายังไม่ตั้งชุดมูฟของซีซั่นนี้ ระบบพาไปหน้าตั้งชุดมูฟก่อน (การตั้งครั้งแรกนี้ไม่นับเป็นการ "ยืนยันปรับชุดมูฟ" ของรอบ)

## 7. การดวล

### 7.1 เลือกผู้ท้าชิง

- **ไม่แสดงตัวเองในรายการ** (ไม่ใช่แค่กดไม่ได้ — ไม่โผล่เลย)
- แสดงเฉพาะคนที่ **ลงสังเวียนแล้ว**
- แสดง **ชื่อ + รูป + อันดับ** ของแต่ละคน · **ไม่แสดงคะแนน** (คะแนนดูที่หน้า ranking หรือหน้าผล)
- มีปุ่ม **🎲 สุ่ม** — สุ่มจากรายการเดียวกัน (จึงไม่มีทางสุ่มโดนตัวเอง)
- **เลือกแล้ว/สุ่มแล้ว เปลี่ยนไม่ได้**
- **Guard:** ถ้าไม่มีใครลงสังเวียนเลยนอกจากตัวเอง → ปุ่มดวลกดไม่ได้ + ข้อความบอกเหตุผล

### 7.2 หน้า VS

จอโชว์ผู้เล่น vs ผู้ท้าชิง (รูป + ชื่อ + อันดับ) พร้อมเอฟเฟกต์ปลุกใจ — **ช็อตลุ้นก่อนเข้าสนาม**

### 7.3 เลือกมูฟ (30 วินาที)

- ผู้เล่นเลือก ค้อน / กรรไกร / กระดาษ → กดยืนยัน
- **นับเวลาถอยหลัง 30 วิ** เริ่มนับเมื่อเข้าหน้านี้
- **หมดเวลา = ระบบสุ่มให้** (สุ่มเท่ากันทั้ง 3 มูฟ) แล้วเข้าสู่ฉากดวลทันที

### 7.4 ฉากเป่ายิ้งฉุบ

- ระบบดึงมูฟของผู้ท้าชิงจากช่องที่ตัวชี้ชี้อยู่
- **ต้องมีช็อตลุ้น** — นับ 1-2-3 แล้วเปิดมูฟพร้อมกันทั้งสองฝั่ง
- เปิดผลแล้วค่อยโชว์คะแนน

## 8. คะแนน

### 8.1 ตารางเรท

| บทบาท | ชนะ | เสมอ | แพ้ |
|---|---|---|---|
| ผู้เล่น — **เลือกเป้าเอง** | **+4** | +1 | **−3** |
| ผู้เล่น — **กดสุ่ม** | **+5** | +1 | **−2** |
| ผู้ท้าชิง | **+3** | +1 | **−2** |
| ดวลนอกรอบ (ทั้งสองฝ่าย) | **+2** | +1 | **−1** |

### 8.2 โบนัสสตรีคชนะต่อเนื่อง

- นับ **เฉพาะตอนเป็นผู้เล่นในเกมหลัก** (ทั้งเลือกเองและกดสุ่ม)
- แพ้/ชนะตอนเป็นผู้ท้าชิง **ไม่นับ ไม่ตัดสตรีค**
- **ดวลนอกรอบไม่นับ ไม่ตัดสตรีค**
- สูตร: ชนะติดต่อกันครั้งที่ `n` → **ตัวคูณ = 1 + (n − 1) × 0.1**

| ชนะติดครั้งที่ | ตัวคูณ | เลือกเอง (ฐาน 4) | สุ่ม (ฐาน 5) |
|---|---|---|---|
| 1 | 100% | 4.0 | 5.0 |
| 2 | 110% | 4.4 | 5.5 |
| 3 | 120% | 4.8 | 6.0 |
| 5 | 140% | 5.6 | 7.0 |
| 10 | 190% | 7.6 | 9.5 |

- **ไม่มีเพดาน**
- **ไม่ชนะเมื่อไหร่ (เสมอหรือแพ้) สตรีคกลับเป็น 0 ทันที** → ครั้งถัดไปที่ชนะคือ "ครั้งที่ 1" (100%)
- **ตัวคูณใช้กับคะแนนชนะเท่านั้น** ไม่ใช้กับเสมอและไม่ใช้กับการหักคะแนนตอนแพ้

### 8.3 พื้นคะแนนและทศนิยม

- ทุกคนเริ่มซีซั่นที่ **30.0**
- **พื้นที่ 0 — คะแนนหลักติดลบไม่ได้** ผลลัพธ์ที่ต่ำกว่า 0 ถูกปัดขึ้นเป็น 0
- อยู่ที่ 0 แล้วแพ้ = ไม่เสียคะแนน **แต่บันทึกสถิติแพ้ตามปกติ** (ไปเจ็บที่ชั้น 3 ของอันดับแทน)
- เก็บ **ทศนิยม 1 ตำแหน่ง** — เรททั้งหมดลงตัวพอดีเสมอ ไม่ต้องปัดเศษ
  > ⚠️ **ข้อควรระวังตอนเขียนโค้ด:** floating point ทำให้ `4 * 1.1 = 4.4000000000000004`
  > → **เก็บคะแนนเป็นจำนวนเต็มหน่วย 0.1 ภายใน** (เช่น 44 = 4.4) แล้วหาร 10 ตอนแสดงผล กันคะแนนเพี้ยนสะสม

### 8.4 ลำดับการคำนวณ (สำคัญ — เขียนโค้ดตามนี้เป๊ะ)

```
1. ตัดสินผล (ชนะ / เสมอ / แพ้) จากมูฟทั้งสองฝั่ง
2. ถ้าผู้เล่นชนะ → streak += 1 ก่อน แล้วคูณ (1 + (streak-1) × 0.1) กับคะแนนฐาน
   ถ้าเสมอหรือแพ้  → streak = 0 (คะแนนไม่คูณ)
3. คำนวณคะแนนผู้เล่น → บวกเข้าคะแนนหลัก → clamp ไม่ให้ต่ำกว่า 0
4. คำนวณคะแนนผู้ท้าชิง → บวกเข้าคะแนนหลัก → clamp ไม่ให้ต่ำกว่า 0
5. อัปเดตสถิติทั้งสองฝ่าย (แยกบทบาท) + อัปเดตเรตการออกมูฟทั้งสองฝ่าย
6. อัปเดตสตรีคสูงสุดที่เคยทำ ถ้าทำลายสถิติเดิม
7. เลื่อนตัวชี้ของผู้ท้าชิงไป 1 ช่อง (วนลูป)
8. บันทึกประวัติการดวล 1 รายการ (เห็นได้ทั้งสองฝ่าย ในประวัติของตัวเอง)
9. จบรอบของผู้เล่น
```

## 9. การจัดอันดับ

| ชั้น | เกณฑ์ | ทิศทาง |
|---|---|---|
| 1 | **คะแนนหลัก** | มากกว่า = ดีกว่า |
| 2 | **คะแนนรอง** (คอลัมน์แยก จำนวนเต็ม) | มากกว่า = ดีกว่า |
| 3 | **จำนวนชนะ − จำนวนแพ้** (เสมอไม่นับ · นับรวมทุกบทบาท) | มากกว่า = ดีกว่า |
| 4 | **จำนวนครั้งที่ลงเป็นผู้เล่น** | มากกว่า = ดีกว่า |
| — | เท่ากันหมด → **อันดับเท่ากันจริง** (อันดับร่วม) | |

- **อันดับร่วมใช้แบบกีฬา:** อันดับ 1 ร่วม 2 คน → คนถัดไปเป็นอันดับ 3 (ไม่มีอันดับ 2)
- คะแนนรอง **ไม่บวกเข้าคะแนนหลักเด็ดขาด** — เจตนาคือให้มันตัดสินเฉพาะตอนเสมอ ไม่ให้ไปเบียดอันดับใคร

## 10. โหมดดวลนอกรอบ

โหมดพิเศษสำหรับกิจกรรม — **ทั้งสองฝ่ายอยู่ตรงนั้น ผลัดกันเลือกมูฟเอง ไม่มีระบบเล่นแทน**

### 10.1 การทำงาน

1. ซุปเลือกผู้เล่น 2 คน (**ไม่ต้องจ่ายเหรียญ** ซุปเปิดได้อิสระ)
2. ส่ง iPad ให้คนที่ 1 เลือกมูฟ → **จอปิดทับ** → ส่งให้คนที่ 2 เลือกมูฟ
3. เปิดผลพร้อมกัน (ช็อตลุ้นเหมือนเกมหลัก)
4. ซุปเลือกวิธีบันทึก

### 10.2 ตัวเลือกตอนจบ

| เลือก | คะแนนหลัก | คะแนนรอง | สถิติแพ้ชนะ | เรตมูฟ |
|---|---|---|---|---|
| **บันทึกเป็นคะแนนหลัก** | ✅ เรทเบา | — | ✅ | ✅ |
| **บันทึกเป็นคะแนนรอง** | — | ✅ เรทเบา (จำนวนเต็ม) | ❌ | ❌ |
| **ไม่บันทึก** | — | — | ❌ | ❌ |

### 10.3 สิ่งที่โหมดนี้ไม่แตะเด็ดขาด

- ❌ **ไม่นับสตรีค** และไม่ตัดสตรีค
- ❌ **ไม่เลื่อนตัวชี้ของทั้งคู่** (ไม่มีใครถูกระบบเล่นแทน)
- ❌ **ไม่นับเป็น "จำนวนครั้งที่ลงเป็นผู้เล่น"** (ชั้น 4 ของอันดับ) — กันคนใช้โหมดนี้ไต่อันดับแทนการจ่ายเหรียญ

## 11. ประวัติและความลับ

### 11.1 จอ "📬 ระหว่างที่คุณไม่อยู่"

โผล่ทุกครั้งที่เข้ารอบ ก่อนเมนูอื่นทั้งหมด แสดง**รายการที่เกิดขึ้นตั้งแต่รอบที่แล้วของเรา**:

- **ใครเป็นคนท้า** (ชื่อ + รูป)
- **มูฟที่ระบบออกแทนเรา**
- **ผลและคะแนนที่ได้/เสียรายครั้ง**
- **สรุปคะแนนรวมช่วงที่ไม่อยู่**
- ⚠️ **ป้ายเตือนเมื่อมีคนไล่เก็บ** — ขึ้นเมื่อมีคนคนเดียวท้าเรา **≥ 3 ครั้ง** ในช่วงนั้น และชนะเรา **เกินครึ่ง**
- ถ้าไม่มีใครท้าเลย → จอบอกแบบสบายใจ ไม่ต้องตกใจ

### 11.2 ประวัติเต็ม

- ดูย้อนหลัง **ได้ตลอดทั้งซีซั่น** จากในรอบของตัวเอง
- **เห็นได้เฉพาะประวัติของตัวเองเท่านั้น** — ประตูกันคือ **ซุปเป็นคนเปิดหน้าให้** ไม่ต้องมีระบบรหัส
- แสดงทั้งบทบาทผู้เล่นและผู้ท้าชิง แยกแท็บได้

### 11.3 ข้อมูลสาธารณะ

- **ranking + คะแนน = ทุกคนดูได้ตลอด** จากหน้าแรก
- record ซีซั่นเก่า = ทุกคนดูได้

## 12. 👑 ภาษีของแชมป์ — เปิดเรตมูฟท็อป 3

> "การเป็นท็อปคือการถูกถอดเสื้อเกราะ หลีกเลี่ยงไม่ได้" — เจ้าของเกม

กดดูได้จากหน้า ranking **โดยทุกคน ตลอดเวลา**

| อันดับ | เปิดอะไร |
|---|---|
| **1** | เรตทั้ง **3 มูฟ** |
| **2 และ 3** | เรตเฉพาะ **มูฟที่ออกบ่อยที่สุด** |

- คิดจาก **จำนวนครั้งที่ออกมูฟทั้งหมดในซีซั่น รวมตอนเป็นผู้ท้าชิง**
- **แสดงจำนวนครั้งกำกับเสมอ** เช่น `✊ ค้อน 45% · จาก 20 ครั้ง` (กันคนอ่าน % หลอกตาตอนฐานน้อย)
- อันดับ 2/3 ถ้ามูฟสูงสุด**เท่ากันหลายอัน → แสดงทั้งหมด**
- ยึด**เลขอันดับที่แสดงบนกระดาน**เสมอ (อันดับ 1 ร่วม 2 คน → เปิด 3 มูฟทั้งคู่ · คนถัดไปเป็นอันดับ 3 เปิดมูฟเดียว)
- **ดวลนอกรอบนับเข้าเรตนี้เฉพาะตอนบันทึกเป็นคะแนนหลัก**
- **อัปเดตสดทุกครั้งที่มีการดวล**

## 14. โครงสร้างข้อมูล (ร่าง)

```ts
type Move = "rock" | "scissors" | "paper";
type DuelOutcome = "win" | "draw" | "lose";

interface Player {
  id: string;              // รหัส 4 ตัว เช่น "A123" — เปลี่ยนไม่ได้
  name: string;
  imageUrl: string;
  moveSet: [Move, Move, Move] | null;   // null = ยังไม่ลงสังเวียนในซีซั่นนี้
  pointerIndex: 0 | 1 | 2;
  mainScoreTenths: number;  // เก็บเป็นหน่วย 0.1 กัน float เพี้ยน · เริ่ม 300 = 30.0
  subScore: number;         // จำนวนเต็ม
  streak: number;           // สตรีคปัจจุบัน (ผู้เล่นเท่านั้น)
  bestStreak: number;
  stats: {
    asPlayer:    { win: number; draw: number; lose: number; rounds: number };
    asChallenger:{ win: number; draw: number; lose: number };
    moveCount:   Record<Move, number>;   // นับทุกครั้งที่ออกมูฟ รวมตอนเป็นผู้ท้าชิง
  };
}

interface DuelRecord {
  id: string;
  at: number;                  // timestamp
  playerId: string;
  challengerId: string;
  wasRandomPick: boolean;
  playerMove: Move;
  challengerMove: Move;
  playerOutcome: DuelOutcome;
  playerDelta: number;         // หน่วย 0.1
  challengerDelta: number;
  streakAfter: number;
  mode: "main" | "offRound";
  offRoundSave?: "main" | "sub" | "none";
}

interface Season {
  id: string;                  // "SS1"
  startedAt: number;
  endedAt: number | null;
  players: Player[];
  duels: DuelRecord[];
}
```

## 16. Edge case ที่ต้องมีเทสคุม

| # | สถานการณ์ | พฤติกรรมที่ถูกต้อง |
|---|---|---|
| 1 | ไม่มีใครลงสังเวียนนอกจากตัวเอง | ปุ่มดวลกดไม่ได้ + บอกเหตุผล |
| 2 | ผู้เล่นยังไม่ตั้งชุดมูฟของซีซั่นนี้ | บังคับตั้งก่อนเข้าเมนูรอบ · ไม่นับเป็นสิทธิ์ปรับชุดมูฟของรอบ |
| 3 | คะแนนอยู่ที่ 0 แล้วแพ้ | คะแนนคงที่ 0 · **สถิติแพ้ถูกบันทึก** |
| 4 | หมดเวลา 30 วิ | ระบบสุ่มมูฟให้ · เข้าฉากดวลทันที |
| 5 | ตัวชี้อยู่ช่อง 3 แล้วถูกท้า | ออกเป่า3 แล้ววนกลับช่อง 1 |
| 6 | ยืนยันปรับชุดมูฟโดยไม่เปลี่ยนค่า | รีเซตตัวชี้ · นับว่าใช้สิทธิ์แล้ว |
| 7 | เข้าหน้าปรับชุดมูฟแล้วกดยกเลิก | ไม่รีเซต · **ไม่นับว่าใช้สิทธิ์** |
| 8 | ดวลนอกรอบแล้วเลือก "ไม่บันทึก" | ไม่มีอะไรเปลี่ยนเลยสักอย่าง |
| 9 | คะแนนเท่ากันทุกชั้น | แสดงอันดับร่วม · คนถัดไปข้ามเลขอันดับ |
| 10 | อันดับ 1 ร่วม 2 คน | เปิดเรต 3 มูฟทั้งคู่ · คนถัดไปเป็นอันดับ 3 เปิดมูฟเดียว |
| 11 | ผู้เล่นออกมูฟ 0 ครั้ง แต่ติดท็อป 3 | แสดง "ยังไม่มีข้อมูล" ไม่ใช่ 0% หรือ NaN |
| 12 | เซฟเสียหาย / localStorage เต็ม | แถบเตือนแบบเกมที่ 1 ไม่พังเงียบ |
| 13 | ลบผู้เล่นที่มีประวัติการดวล | ประวัติเก่ายังอ้างชื่อได้ (เก็บ snapshot ชื่อไว้ใน DuelRecord) |

## 17. สิ่งที่ตัดทิ้งแล้ว (อย่าเผลอใส่กลับ)

| ตัดทิ้ง | เหตุผล |
|---|---|
| คูลดาวน์ห้ามท้าคนเดิมซ้ำ | ผู้เล่น 10+ คน และปุ่มสุ่มกระจายเป้าอยู่แล้ว |
| ตัวชี้มีโอกาสหลุดมูฟ (สุ่มแทนเซต) | **ฆ่าแรงจูงใจจ่ายเหรียญเปลี่ยนชุดมูฟ = ฆ่าท่อยอดขาย** |
| เพดานสตรีค | ความต่อเนื่องคือความท้าทาย |
| ตัวคูณ ×2 ในเกณฑ์รอง | ไม่จำเป็นแล้วเมื่อแยก "จำนวนครั้งที่เป็นผู้เล่น" เป็นชั้นล่าง |
| สตรีคสูงสุด / ใครถึงคะแนนก่อน เป็นเกณฑ์อันดับ | เจ้าของเกมเคาะให้เหลือ 2 ชั้นสถิติพอ (สตรีคสูงสุดยังโชว์เป็นสถิติเท่ๆ อยู่) |
| ระบบกระเป๋าเหรียญในแอป | เหรียญอยู่กับซุปในโลกจริง · แอปนับจำนวนรอบให้แทน |
| ระบบรหัสผ่านดูประวัติ | ซุปเป็นด่านกันเองอยู่แล้ว |
