# ฉากปะทะ VS + แก้ศัพท์เกม — Design (2026-07-21)

เกมที่ 3 "เป่า ยิ้ง ฉุบ! อารีน่า!" · เจ้านายเคาะผ่านแล้วก่อนลงมือ

---

## 1. เป้าหมาย

1. เพิ่ม **ฉากปะทะ VS 5 วินาที** คั่นก่อนเป่ายิ้งฉุบ ให้เหมือนเกมต่อสู้จริง
2. แก้ **ศัพท์เกม** ให้ตรงนิยามของเจ้านาย ทั้งข้อความบนจอ ชื่อตัวแปรในโค้ด และเอกสาร

## 2. ศัพท์ใหม่

| ความหมาย | คำไทย | ชื่อในโค้ด | ชื่อเดิม (v1) |
|---|---|---|---|
| ทุกคนในเกม | ผู้เล่น | `player` | `player` |
| คนจ่ายเหรียญมาท้า (ฝ่ายรุก) | **ผู้ท้าชิง** | `challenger` | `player` |
| คนถูกท้า (ฝ่ายรับ) | **คู่แข่ง** | `opponent` | `challenger` |

ชื่อเก่ากับใหม่ **กลับข้างกัน** จึงต้องแปลงเซฟเก่า ไม่งั้นประวัติ/สถิติจะอ่านผิดฝั่งแบบเงียบๆ

**Migration:** `SAVE_VERSION` 1 → 2 · `migrateFromV1()` ใน `state/storage.ts` แปลง
`duels[]` (ทุกฟิลด์), `players[].stats` (`asPlayer`→`asChallenger`, `asChallenger`→`asOpponent`),
`round.playerId`→`challengerId`, `config.challengerRates`→`opponentRates`
อ่านค่าเดิมทั้งชุดก่อนแล้วค่อยเขียน — กันเขียนทับกันเองตอนสลับชื่อ

## 3. ฉากปะทะ VS

**ลำดับใหม่:** เลือกคู่แข่ง → เลือกมูฟ (30 วิ) → **VS ปะทะ (5 วิ)** → เป่ายิ้งฉุบ → ผลดวล

| เวลา | ขั้น (`Stage`) | เกิดอะไร |
|---|---|---|
| 0–0.6 วิ | `in` | สองฝั่งพุ่งเข้าจากนอกจอ มีเส้นสปีดวิ่ง |
| 0.6 วิ | `clash` | จอสั่น + แฟลชขาว + ประกายปะทะ + ป้าย VS กระแทกลง |
| 0.9–3.2 วิ | `info` | อันดับ/สถิติ/สตรีค ไล่ขึ้น + แถบสถิติเจอกันของคู่นี้ |
| 3.2–5.0 วิ | `ready` | ป้าย "พร้อม!" เต้น แล้วตัดเข้าเป่ายิ้งฉุบ |

- ซ้าย = ผู้ท้าชิง (มุมน้ำเงิน) · ขวา = คู่แข่ง (มุมชมพู) · เฉือนทแยงตรงรอยต่อ
- **แตะที่ไหนก็ข้ามได้** + ปุ่ม "ข้าม →" · กันเด้งซ้ำด้วย `done` ref (กดข้ามพร้อมหมดเวลา = เรียก `onReady` ครั้งเดียว)
- `prefers-reduced-motion` → ย่นเหลือ 1.4 วิ ตัดพุ่ง/สั่น/ไฟวิ่งทิ้ง
- เว้นจุดต่อเสียงไว้ (`TODO(sound)`) — ระบบเสียงเป็นงานแยก

## 4. วิธี verify

- `npm run test` — เทสหน่วย รวมเทส migration เซฟ v1 → v2 (ข้อมูลไม่หาย ไม่แปลงซ้ำ)
- `node verify-flow.mjs` — เดินทั้ง flow บนจอ iPad วัด overflow ทุกจอ + เช็คฉาก VS ครบ 3 ช่วง
  (มีแถบสถิติเจอกัน · ขึ้นป้าย "พร้อม!" · ปุ่มข้ามกดได้)
